Simple API

Terse and Expressive

SceneJS provides easy access to WebGL through a simple and declarative JavaScript API. While most scene graph frameworks are object-oriented, SceneJS is functional, which enables its scene definitions to be really compact and expressive, while hooking into other JavaScript code just that little bit more smoothly.

Demo: Hello Teapot!

Tutorial: A Bare Bones OpenGL Teapot

Efficiency

Memoization, Automatic Shader Generation and VRAM Management

SceneJS uses WebGL efficiently for you, quietly behind the scenes. It caches aggressively to avoid recalculation, automatically generates optimal shaders, intelligently swaps resources like shaders, textures and vertex buffers in and out of video RAM, and avoids needless repetition of costly WebGL buffer loads by performing them lazily.

Benchmark demo: 300 Teapots

Distributed Content

Cross-Domain Components

SceneJS is geared to composition of scenes from re-usable, parameterised fragments. It automatically manages a lean set of in-memory fragments by loading them cross-domain on demand, while evicting them following a least-recently-used policy to maintain the pool of free memory. Its terse API means that fragments are compact, allowing them to propagate rapidly across the network. Being JavaScript, they evaluate straight into the browser with no expensive parsing.

Demo: Cross-Domain Import of a Teapot

Scene Optimisation

Frustum Culling

SceneJS provides a flexible frustum culling mechanism whereby you may enclose subgraphs in bounding volumes, so that they are only processed when their volumes intersects the view volume.

Tutorial: Frustum Culling with Bounding Boxes


Level-of-Detail Culling

Level-of-detail (LOD) selection enhances rendering efficiency by decreasing the complexity of a 3D object representation as it moves away from the viewer. SceneJS provides a very flexible LOD mechanism. For a 3D object, a set of subgraphs, each representing the object at a different level of detail, can be wrapped in a bounding volume so that the approprate subgraph is rendered for the current projected canvas size of the bounding volume.


Demo: Basic Level-of-Detail Demo

Generatable

Procedural Generation

Procedural generation involves defining scene content algorithmically rather than manually. SceneJS provides the ability to wrap a subgraph in a function that will loop over it, effectively instancing it multiple times within each frame, each time in a different state.

Demo: Generating a Spiral Staircase

Animatable

Dynamic Parameterisation

As you saw back in the Hello Teapot example, most scene nodes can be configured with functions that evaluate their parameters at render-time.

Scalar Interpolation

Almost any scalar property in SceneJS can be animated by linear, cosine, cubic or constant interpolation within sequences of key values. That includes transformations, materials, texture, lighting, even other interpolations, and is usually done as a function of time.

Demo: Rotating a Teapot using Interpolators
