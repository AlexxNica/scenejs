SceneJS

Succinct and expressive
scene definition for WebGL

SceneJS is a JavaScript framework by Lindsay Kay that provides a functional API through which you can easily create interactive 3D scenes on the WebGL canvas element.

General Syntax

Below is a simple example that renders the venerable OpenGL teapot shown to the right.

To render the teapot, SceneJS will traverse the scene in depth-first order. Each node is a function that will set some scene state on entry, then un-set it again before exit. In this graph, the scene node activates a WebGL Canvas element, a perspective defines the projection, a lookAt defines the camera, lights defines a light source, rotate tilts the model space, material sets some colours, and objects.teapot draws our teapot.

Note how node declarations generally have a configuration object followed by zero or more children:

node({ foo: "bar" }, node( .. ), node( .. ) ... )

For brevity, the configuration objects need only specify values where defaults are overridden.


Take a closer look at that rotate node. See how it takes a function which creates its configuration object? You can do that when you want a node's configuration to be evaluated dynamically at traversal-time. The function takes an immutable data object, which is SceneJS's mechanism for passing variables down into a scene graph. Using the angle property on that data, the function creates a configuration that specifies a rotation about the X-axis. See also how we inject that angle when we render the scene.


To really get an idea of what SceneJS can do, take a look at the features list, which grows a little with every release.
