Scene Graph Branching - Teapot Selector

Although SceneJS manages WebGL state settings for you automatically, the SceneJS.Renderer node allows you to set them explicitly for portions of your scene graph, perhaps to optimise its performance or do some special rendering effect.

Use it to explicitly set things for subgraphs, such as the current viewport extents, scissoring, enable/disable shading, backface culling, and so on.

Normally you wouldn't need to use renderer nodes, but you can insert them throughout your scene when you want more control over WebGL, perhaps to optimise your scene graph.

They can also be nested, where the inner renderers will override the effects of the outer ones.

This example is taken from one of the COLLADA test examples, but with a renderer node inserted with all its properties set to defaults. When you omit a renderer node, SceneJS will effectively provide this renderer by default.
