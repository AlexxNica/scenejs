1 /**
  2  * @class Causes an instance of a resource to be inserted into a scene graph.
  3  *
  4  * <p>The resource can come from the entire contents of a model file, a selected element in a model file, or from a
  5  * Symbol that has been previously rendered in the scene graph. For each of these cases, the Instance's uri parameter
  6  * will take a different form, as shown in these examples.</p>
  7  *
  8  * <p>To instance an entire model from a file, specify an absolute URI to the file with no fragment part. Here we're
  9  * instancing the default scene defined in a COLLADA file:</p>
 10  * <pre><code>
 11  * new SceneJS.Instance({
 12  *    uri: "http://foo.com/models/airplane.dae"
 13  * })
 14  * </code></pre>
 15  * <p>For some file formats, SceneJS allows you to instance a selected a resource (or "asset") from within them. Here
 16  * we're instancing a selected scene within a COLLADA file:</p>
 17  * <pre><code>
 18  * new SceneJS.Instance({
 19  *     uri: "http://foo.com/models/suburbia.dae#house_scene"
 20  * })
 21  * </code></pre>
 22  * <p>If we know that file is really big and will take longer than the default of 180 seconds to arrive, we can configure
 23  * our own timeout, in this case a generous six minutes:</p>
 24  *  <pre><code>
 25  * new SceneJS.Instance({
 26  *     uri: "http://foo.com/models/suburbia.dae#house_scene",
 27  *     loadTimeoutSecs: 360
 28  * })
 29  * </code></pre>
 30  * <p>When a Symbol has been rendered prior to the Instance during scene traversal, then the Instance may instantiate it
 31  * with a URI that contains only the fragment part, which will walk the SIDs (sub-identifiers) of previously rendered
 32  * nodes to resolve the Symbol. Here's the most trivial case - see how the fragment URI of the instance maps to the
 33  * Symbol's SID.<p>
 34  * <pre><code>
 35  * new SceneJS.Symbol({ sid: "myBox" },
 36  *    new SceneJS.objects.Cube()
 37  * ),
 38  *
 39  * new SceneJS.Instance( { uri: "myBox" })
 40  * </code></pre>
 41  * <p>Another case, in which an Instance resolves a Symbol that is within a namespace defined by the SID of a parent
 42  * node:</p>
 43  * <pre><code>
 44  * new SceneJS.Node({ sid: "mySymbols" },
 45  *     new SceneJS.Symbol({ sid: "myBox" },
 46  *         new SceneJS.objects.Cube()
 47  *     )
 48  * ),
 49  *
 50  * new SceneJS.Instance( { uri: "mySymbols/myBox" })
 51  * </code></pre>
 52  * <p>Think of these nested SIDs as directories, where the URI fragment part works works the same way as a directory
 53  * path. An absolute fragment path begins with a '/' - in the following example, if the node with the "moreSymbols" SID
 54  * is defined at the top level, we may then reference it with an absolute path:</p>
 55  * <pre><code>
 56  * new SceneJS.Node({ sid: "moreSymbols" },
 57  *     new SceneJS.Symbol({ sid: "myOtherBox" },
 58  *         new SceneJS.objects.Cube()
 59  *     )
 60  * ),
 61  *
 62  * new SceneJS.Instance( { uri: "/moreSymbols/myOtherBox" })
 63  * </code></pre>
 64  *
 65  * <p>For some model file formats, you can use an absolute URI with a fragment to instantiate hierarchically-organised
 66  * resources within them. When SceneJS loads a COLLADA model, for example, it will create Symbol nodes within SID
 67  * namespaces (as shown above) that will allow you to instantiate selected scenes. You can even instantiate a
 68  * selected COLLADA scene via one of its cameras: </p>
 69  *
 70  * </code></pre>
 71  * new SceneJS.Instance( {
 72  *     uri: "http://foo.com/models/suburbia.dae#house_scene/camera1"
 73  * });
 74  *
 75  * <h2>File Loading States</h2>
 76  * <p>When instancing content from a file, an Instance node must first perform an HTTP GET to load it. That is, unless
 77  * some other Instance has already loaded that file, in which case SceneJS will have internally cached a subgraph
 78  * parsed from that file.</p>
 79  *
 80  * <p>When instancing content from a file, an Instance node must first perform an HTTP GET to load it. That is, unless
 81  * some other Instance has already loaded that file, in which case SceneJS will have internally cached a subgraph
 82  * parsed from that file.</p>
 83  *
 84  * <p>Each Instance that must load a file will transition through six different states as its request is processed,
 85  * which may be queried through the Instance's {@link #getState} method:</p>
 86  * <ul><li>{@link #STATE_INITIAL} in which it is about to make an asynchronous request for the content.</li>
 87  * <li>{@link #STATE_LOADING} in which it has made the request and is awaiting the response, which it will receive
 88  * at any time, either within a scene render or between scene renders. In this state it has nothing to render
 89  * and will pass scene traversal on to its next sibling.</li>
 90  * <li>{@link #STATE_READY} in which it has successfully received its content and parsed it into a subgraph.
 91  * From here the node will transition to {@link #STATE_RENDERED} as soon as the subgraph has been rendered
 92  * and traversal has arrived back at this node on the way back up the graph. It may instead transition back
 93  * to {@link #STATE_INITIAL} as soon as it has not been rendered in a while and has then had its subgraph destroyed
 94  * by SceneJS to reclaim memory.</li>
 95  *  * <li>{@link #STATE_RENDERING} in which it has successfully loaded and parsed its target file or resolved its
 96  * target symbol and is about to render (instance) that target. From here the node will transition to
 97  * {@link #STATE_RENDERED} as soon as rendered its target.</li>
 98  * <li>{@link #STATE_RENDERED} in which it has successfully rendered its target at least once. If the target is a file,
 99  * then from here the node will transition back to {@link #STATE_INITIAL} as soon as the target has not been rendered in
100  * a while and has been destroyed by SceneJS to reclaim memory.</li>
101  * <li>{@link #STATE_FREED} in which it has been dormant too long and SceneJS has reclaimed memory by destroying
102  * the subgraph. When next rendered, it will then repeat its load request and transition
103  * to {@link #STATE_LOADING}.</li>
104  * <li>{@link #STATE_ERROR} if SceneJS has permanently inactivated the node after load failed</li></ul>
105  * <p>Note that when instantiating from a file that has been previously loaded by another Instance, it will transition
106  * directly to {@link #STATE_RENDERED}.<p>
107  *
108  * <p><b>Loading Cross-Domain</b></p>
109  * <p>When SceneJS is configured with a JSONP strategy via {@link SceneJS.setJSONPStrategy}, you can perform the
110  * load cross-domain. Otherwise, the URL of the content must be at the same domain as the scene definition's JavaScript file
111  * in order to not violate the browser' same-domain security policy.
112  *
113  * <p><b>Live Examples</b></p>
114  * <li><a target = "other" href="http://bit.ly/scenejs-asset-load">Example 1</a></li>
115  * </ul>
116  *
117  * @events
118  * @extends SceneJS.Node
119  * @constructor
120  * Creates a new SceneJS.Instance
121  *  @param {Object} [cfg] Static configuration object
122  * @param {String} cfg.uri URI of file to load
123  * @param {int} [cfg.timeoutSecs] Timeout - falls back on any loadTimoutSecs that was configured on the {@link SceneJS.Scene}
124  * at the root of the scene graph, or the default 180 seconds if none configured there
125  * @param {function(SceneJS.Data):Object} [fn] Dynamic configuration function
126  * @param {...SceneJS.Node} [childNodes] Child nodes
127  */
128 SceneJS.Instance = function() {
129     SceneJS.Node.apply(this, arguments);
130     this._nodeType = "instance";
131     this._uri = null;
132     this._symbol = null;
133     this._state = SceneJS.Instance.STATE_INITIAL;
134     if (this._fixedParams) {
135         this._init(this._getParams());
136     }
137 };

141 /**
142  * Initial state of a remote Instance, in which it has not been rendered yet and thus loading of its target file has not
143  * yet begun. From here a remote Instance will transition to {@link #STATE_LOADING} as soon as it is first rendered.
144  * @const
145  */
146 SceneJS.Instance.STATE_INITIAL = 0;
147 
148 /**
149  * State of a local or remote-mode Instance in which instantiation has failed. For a remote instance this can be either
150  * due to HTTP error (eg. HTTP 404) or failure to parse the response; in the assumption that the file is unavailable for
151  * the remainder of the lifetime of the scene graph, a remote Instance will then remain fixed in this state. For a local
152  * instance, this condition might be temporary (eg. the target Symbol is just not being rendered yet for some reason),
153  * so a local Instance will then try instancing again when next rendered, transitioning to {@link STATE_RENDERING} when
154  * that succeeds.
155  * @const
156  */
157 SceneJS.Instance.STATE_ERROR = -1;
158 
159 /**
160  * State of a local or remote Instance in which it will attempt to instantiate its target when next rendered. For a
161  * remote Instance, this is when it has successfully loaded its target file and parsed it into a subgraph. For a local
162  * Instance, this is when it is ready to attempt aquisition of its target Symbol. From here, a remote node will
163  * transition to {@link #STATE_RENDERING} and instantiates its target when next rendered. A local Instance will do
164  * the same when next rendered, providing it succeeds in finding its target Symbol, otherwise the local Instance will
165  * transition to {@link #STATE_ERROR}.
166  * target.
167  * @const
168  */
169 SceneJS.Instance.STATE_READY = 2;
170 
171 
172 /**
173  * State of a local or remote Instance in which it is rendering its target. While in this state, you can
174  * obtain its target subgraph through its getTarget() method. From this state, the Instance will transition to
175  * {@link #STATE_READY} once it has completed rendering the target.
176  * @const
177  */
178 SceneJS.Instance.STATE_RENDERING = 3;

189 // @private
190 SceneJS.Instance.prototype._init = function(params) {
191     if (params.uri) {
192         this._uri = params.uri;
193         this._state = SceneJS.Instance.STATE_READY;
194         this._mustExist = params.mustExist;
195     }
196 };

198 // @private
199 SceneJS.Instance.prototype._render = function(traversalContext, data) {
200     if (!this._fixedParams) {
201         this._init(this._getParams(data));
202     }
203     if (this._uri) {
204         this._instanceSymbol(this._uri, traversalContext, data);
205     } else {
206         throw SceneJS._errorModule.fatalError(
207                 new SceneJS.InvalidNodeConfigException(
208                         "SceneJS.instance uri property not defined"));
209     }
210 };

212 // @private
213 SceneJS.Instance.prototype._changeState = function(newState, exception) {
214     var oldState = this._state;
215     this._state = newState;
216     if (this._listeners["state-changed"]) { // Optimisation
217         this._fireEvent("state-changed", { oldState: oldState, newState: newState, exception : exception });
218     }
219 };

221 /* Returns a traversal context for traversal of the children of the given target node.
222  *
223  * If this Instance has children then it will have a callback that will render them after the last of
224  * the target's sub-nodes have rendered, as effectively the children of that last node. The callback will
225  * create a traversal context for the sub-nodes that will:
226  *
227  * - initially flag the traversal as inside the right fringe if the there are more than one child
228  * - pass on any callback that was passed in on the traversal context to this Instance
229  * - pass on any WithConfigs configs that were passed in on the traversal context to this Instance
230  */
231 SceneJS.Instance.prototype._createTargetTraversalContext = function(traversalContext, target) {
232     var callback;
233     this._superCallback = traversalContext.callback;
234     var _this = this;
235     if (!this._callback) {
236         this._callback = function(traversalContext, data) {
237             var subTraversalContext = {
238                 callback : _this._superCallback,
239                 insideRightFringe : _this._children.length > 1,
240                 configs: traversalContext.configs,
241                 configsModes: traversalContext.configsModes
242             };
243             _this._renderNodes(subTraversalContext, data);
244         };
245     }
246     return {
247         callback: this._callback,
248         insideRightFringe:  target._children.length > 1,
249         configs: traversalContext.configs,
250         configsModes: traversalContext.configsModes
251     };
252 };
253 
254 /**
255  * Recursively renders the subtrees (child nodes) of the Instance in left-to-right, depth-first order. As the recursion
256  * descends, the traversalContext tracks whether the current node is inside the right fringe of the right-most subtree.
257  * As soon as the current node is at the right fringe and has no children, then it is the last node to render among all
258  * the sub-nodes (the "terminal node"). If a callback is then present on the traversalContext, then that means that this
259  * Instance is actually within a subtree of a Symbol node that is being instanced by another super-Instance. The
260  * callback is then invoked, which causes the traversal of the super-Instance's subtree as if it were a child of the
261  * terminal node.
262  *
263  * @param traversalContext
264  * @param data
265  */
266 SceneJS.Instance.prototype._renderNodes = function(traversalContext, data) {
267     var numChildren = this._children.length;
268     var child;
269     var childConfigs;
270     var configUnsetters;
271 
272     if (numChildren == 0) {
273 
274         /* Instance has no child nodes - render super-Instance's child nodes
275          * through callback if one is passed in
276          */
277         if (traversalContext.callback) {
278             traversalContext.callback(traversalContext, data);
279         }
280 
281     } else {
282 
283         /* Instance has child nodes - last node in Instance's subtree will invoke
284          * the callback, if any (from within its SceneJS.Node#_renderNodes)
285          */
286         for (var i = 0; i < numChildren; i++) {
287             child = this._children[i];
288 
289             childConfigs = traversalContext.configs;
290             configUnsetters = null;
291 
292             if (childConfigs && child._sid) {
293                 childConfigs = traversalContext.configs["#" + child._sid];
294                 if (childConfigs) {
295                     configUnsetters = this._setConfigs(childConfigs, child);
296                 }
297             }
298 
299             var childTraversalContext = {
300                 insideRightFringe : (i < numChildren - 1),
301                 callback : traversalContext.callback,
302                 configs : childConfigs || traversalContext.configs,
303                 configsModes : traversalContext.configsModes
304             };
305 
306             child._render.call(child, childTraversalContext, data);
307 
308             if (configUnsetters) {
309                 this._unsetConfigs(configUnsetters);
310             }
311         }
312     }
313 };

315 /** Instances a Symbol that is currently defined after being rendered prior to this Instance
316  *
317  * @private
318  * @param symbolSIDPath Path to Symbol, relative to this Instance
319  * @param traversalContext
320  * @param data
321  */
322 SceneJS.Instance.prototype._instanceSymbol = function(symbolSIDPath, traversalContext, data) {
323     this._symbol = SceneJS._instancingModule.acquireInstance(symbolSIDPath);
324     if (!this._symbol) {
325         //SceneJS._loggingModule.info("SceneJS.Instance could not find SceneJS.Symbol to instance: '" + symbolSIDPath + "'");
326         if (this._mustExist) {
327             SceneJS._instancingModule.releaseInstance();
328             throw SceneJS._errorModule.fatalError(
329                     new SceneJS.SymbolNotFoundException
330                             ("SceneJS.Instance could not find SceneJS.Symbol to instance: '" + symbolSIDPath + "'"));
331         }
332         this._changeState(SceneJS.Instance.STATE_ERROR);
333     } else {
334         this._changeState(SceneJS.Instance.STATE_RENDERING);
335         this._symbol._renderNodes(this._createTargetTraversalContext(traversalContext, this._symbol), data);
336         SceneJS._instancingModule.releaseInstance();
337         this._changeState(SceneJS.Instance.STATE_READY);
338         this._symbol = null;
339     }
340 };
