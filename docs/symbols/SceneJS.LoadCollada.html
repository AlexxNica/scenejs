Class SceneJS.LoadCollada


Extends SceneJS.Load.
A scene node that asynchronously loads its subgraph from a COLLADA file.

This node is configured with the location of the COLLADA file. When first visited during scene traversal, it will begin the load and allow traversal to continue at its next sibling node. When on a subsequent visit its subgraph has been loaded, it will then allow traversal to descend into that subgraph to render it.

Loading Cross-Domain

When SceneJS is configured with a JSONP strategy via SceneJS.setJSONPStrategy, you can perform the load cross-domain. Otherwise, the URL of the content must be at the same domain as the scene definition's JavaScript file in order to not violate the browser' same-domain security policy.

Monitoring Load Progress

You can monitor loads by registering "process-started" and "process-killed" listeners with SceneJS.addListener().

    • Usage Example

    The SceneJS.LoadCollada node shown below loads a target asset from the with ID "propeller" in a Collada file "airplane.dae" stored on the local domain.

    
var exampleScene = new SceneJS.Scene({

      new SceneJS.LoadCollada({
                 uri: "airplane.dae",
                 node: "propeller"
           })
 );

    {int} getConfig()
    Returns an object (owned by the SceneJS.LoadCollada node) through you can access the configuration of various nodes (such as lights and cameras) that the SceneJS.LoadCollada has created from the Collada file, along with some read-only contributor data on the file.

    You can walk the object to find out what nodes were created, and set properties within it to configure their state when they are next rendered. Each camera and light will have a unique ID. The object provides a property through which you can select which camera is currently active (see below).

    Various sub-objects provide data to dynamic configuration functions on the nodes (see how those functions work in SceneJS.Node). Therefore, the object works as efficient read/write access to their render-time configuration.

    Since the object is generated while creating nodes as the Collada file is loaded, this function will only return it when the SceneJS.LoadCollada's state is SceneJS.Load.STATE_LOADED or SceneJS.Load.STATE_RENDERED, which you can check with a prior call to SceneJS.Load#getState.

    Shown below is the config object's general structure. When cameras are loaded, it contains an array of camera configurations, each with properties pertaining to the camera type, plus a configuration for a default camera. When lights are loaded, it contains an array of light configurations, again with pertenant properties for each of those. When caneras are loaded, the activeCamera property may be set to a camera's ID to activate it.

    
{
  contributor: {
      author        : "Jane Doe",
      authoringTool : "",
      comments      : String,
      copyright     : String,
      sourceData    : String
  },

  // Set this to the ID of one of the cameras (see below) to activate it.

  // When undefined

  activeCamera: String,

  // A configuration for each  parsed:

  cameras: [
      {
          // Uniquely identifies camera in .dae file

          id: String,


          // Camera projection configuration. There are two types, each of which
          // will have different properties.

          optics: {

              type: String,       // "ortho" or "perspective"


              // Properties when optics type is "ortho":

              left: Number,
              right: Number,
              bottom: Number,
              top: Number,
              near: Number,
              far: Number,


              // Properties when optics type is "perspective":

              fovy: Number,
              aspect: Number,
              near: Number,
              far: Number
          },


          // When the camera is instanced by a  transform, there will
          // also be a lookat property:

          lookat: {
              eye: {
                  x: Number,
                  y: Number,
                  z: Number
              },

              look: {
                  x: Number,
                  y: Number,
                  z: Number
              },

              up: {
                  x: Number,
                  y: Number,
                  z: Number
              }
          }
      }
  ],

  // The default camera which will be active when no  elements
  // were loaded, or when activeCamera does not resolve to any of the
  // loaded cameras' IDs:

  defaultCamera : {
      lookat: {
          eye: { x: 0, y: 0, z: 1},
          look: { x: 0, y: 0, z: 0 },
          up: { x: 0, y: 1, z: 0}
      },
      optics: {
          type: "perspective",
          fovy : 60.0,
          aspect : 1.0,
          near : 0.10,
          far : 10000.0
      }
  },

  // A configuration for each . Each light configuration
  // identifies it's type, for which it will have the pertenant
  // properties.

  lights: [

      // A point light:

      {
          type: "point",
          pos: { x: Number, y: Number, z: Number },
          color: { r : Number, g: Number, b: Number },
          constantAttenuation : Number,
          linearAttenuation : Number,
          quadraticAttenuation : Number
      },

      // A directional light:

      {
          type: "dir",
          dir: { x: Number, y: Number, z: Number },
          color: { r : Number, g: Number, b: Number }
      }
  ]
}
