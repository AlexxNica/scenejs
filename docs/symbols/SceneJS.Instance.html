Class SceneJS.Instance


Extends SceneJS.Node.
Instantiates the subtree of a target SceneJS.Symbol at the node's location within the scene graph. The flexible Instancing Algorithm also permits recursive instantiation, where target SceneJS.Symbols may define further instances of other target SceneJS.Symbols, and so on. Instances may also be parameterised using the data flow capabilities provided by the SceneJS.WithData and SceneJS.WithConfigs nodes.

When a SceneJS.Symbol has been rendered prior to the Instance during the current scene traversal, then the Instance can instantiate it with a URI that which will walk the SIDs (sub-identifiers) of previously rendered nodes to resolve the SceneJS.Symbol. Here's the most trivial case - see how the fragment URI of the instance maps to the SceneJS.Symbol's SID.


new SceneJS.Symbol({ sid: "myBox" },
   new SceneJS.objects.Cube()
),

new SceneJS.Instance( { uri: "myBox" })

Another case, in which an Instance resolves a Symbol that is within a namespace defined by the SID of a parent node:


new SceneJS.Node({ sid: "mySymbols" },
    new SceneJS.Symbol({ sid: "myBox" },
        new SceneJS.objects.Cube()
    )
),

new SceneJS.Instance( { uri: "mySymbols/myBox" })

Think of these nested SIDs as directories, where the URI fragment part works works the same way as a directory path. An absolute fragment path begins with a '/' - in the following example, if the node with the "moreSymbols" SID is defined at the top level, we may then reference it with an absolute path:


new SceneJS.Node({ sid: "moreSymbols" },
    new SceneJS.Symbol({ sid: "myOtherBox" },
        new SceneJS.objects.Cube()
    )
),

new SceneJS.Instance( { uri: "/moreSymbols/myOtherBox" })

States and Events

A SceneJS.Instance has four states which it transitions through during it's lifecycle, as described below. After it transitions into each state, it will fire an event - see SceneJS.Node. Also, while in #STATE_RENDERING, it can provide its target SceneJS.Symbol node via #getTarget.

<static> SceneJS.Instance.STATE_READY
State of a SceneJS.Instance in which it will attempt to instantiate its target when next rendered. This is when it is ready to attempt aquisition of its target SceneJS.Symbol. From here, it will transition to #STATE_RENDERING if that succeeds, otherwise it will transition to #STATE_ERROR.
<static> SceneJS.Instance.STATE_RENDERING
State of an SceneJS.Instance in which it is currently rendering its target SceneJS.Symbol. While in this state, you can obtain the target SceneJS.Symbol through #getTarget. From this state, the SceneJS.Instance will transition back to #STATE_READY once it has completed rendering the target.
